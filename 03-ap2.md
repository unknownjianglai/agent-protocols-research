# AP2 (Agent Payments Protocol) 深度调研报告

## 概述

**Agent Payments Protocol (AP2)** 是由 **Google** 开发的开放协议，专门为新兴的**智能体经济**提供支付基础设施。AP2旨在让AI智能体能够自主、安全地完成支付交易，是智能体经济时代的"支付基座"。

> **发布时间**: 2025年  
> **主要推动方**: Google Cloud  
> **协议类型**: 支付协议  
> **开源状态**: 开源

---

## 核心愿景

AP2的愿景是构建一个**智能体原生的支付基础设施**，支持：

1. **机器对机器支付** - AI智能体之间的自主交易
2. **人机协作支付** - 人类与AI智能体的混合支付场景
3. **可信自动化** - 在保持安全可控的前提下实现支付自动化

## 核心创新: Mandate (授权凭证)

AP2的核心创新是引入了**Mandate（授权凭证）**机制，这是一种可编程的支付授权方式。

### 三种核心Mandate类型

```
┌─────────────────────────────────────────────────────────────┐
│                 AP2 Mandate 类型体系                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐ │
│  │ Cart Mandate    │  │ Intent Mandate  │  │ Open Mandate│ │
│  │ 购物车授权       │  │ 意图授权         │  │ 开放授权     │ │
│  └────────┬────────┘  └────────┬────────┘  └──────┬──────┘ │
│           │                    │                   │        │
│           ▼                    ▼                   ▼        │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐ │
│  │ • 指定商品       │  │ • 购买意图       │  │ • 灵活额度   │ │
│  │ • 固定金额       │  │ • 预算上限       │  │ • 时间限制   │ │
│  │ • 一次性支付     │  │ • 条件触发       │  │ • 场景限制   │ │
│  └─────────────────┘  └─────────────────┘  └─────────────┘ │
│                                                             │
│  场景: 具体商品购买   场景: 智能推荐购买   场景: 订阅/定期购买 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Mandate 详细说明

| Mandate类型 | 描述 | 适用场景 | 安全级别 |
|------------|------|---------|---------|
| **Cart Mandate** | 针对特定购物车的授权 | 用户明确指定商品 | 最高 |
| **Intent Mandate** | 基于购买意图的授权 | "买一双跑鞋"这类模糊需求 | 中等 |
| **Open Mandate** | 开放额度授权 | 订阅服务、定期购买 | 可控 |

## 实际应用案例

### 案例1: 公益捐赠智能体

使用Google ADK和AP2构建的慈善捐赠智能体：

```
用户: "我想给儿童教育项目捐款100元"
      ↓
Agent: "我找到了3个符合条件的项目，这是详情..."
      ↓
用户: "选择第二个"
      ↓
Agent: [创建Intent Mandate]
      ↓
用户: 确认授权
      ↓
Agent: [通过AP2完成捐赠]
      ↓
用户: "给我捐赠证书"
      ↓
Agent: [生成并发送证书]
```

### 案例2: 企业采购Agent

企业内部使用的智能采购助手：

```
员工: "申请采购5台显示器，预算5000元以内"
      ↓
采购Agent: [创建Open Mandate，额度5000元]
      ↓
主管: 审批通过
      ↓
采购Agent: [搜索供应商] → [比价] → [选择最优]
      ↓
采购Agent: [使用Mandate完成支付]
      ↓
系统自动: 生成采购订单、更新库存、通知财务
```

## 优势与挑战

### 优势

1. **智能体原生设计**: 专为AI智能体支付场景设计
2. **灵活授权机制**: Mandate系统支持多种授权模式
3. **安全可控**: 细粒度的权限控制和审计
4. **与UCP协同**: 完整的商务+支付解决方案

### 挑战

1. **用户教育**: 需要让用户理解Mandate概念
2. **信任建立**: 用户需要信任AI智能体的支付决策
3. **监管合规**: 需要符合各地区支付监管要求

## 与x402的对比

| 维度 | AP2 | x402 |
|------|-----|------|
| **主要推动方** | Google | Coinbase |
| **支付模式** | 基于凭证授权 | 基于HTTP 402 |
| **结算方式** | 传统支付+数字支付 | 区块链/稳定币 |
| **适用场景** | 电商、服务采购 | API调用、微支付 |
| **技术栈** | HTTPS/REST | HTTP/区块链 |
| **去中心化** | 中心化协调 | 去中心化 |
